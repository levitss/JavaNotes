내부 클래스의 정의 
    클래스 내부에 선언한 클래스 
    이 클래스를 감싸고 있는 외부 클래스와 밀접한 연관이 있는  경우가 많음
    다른 외부 클래스에서 사용할 일이 없는 경우에 내부 클래스로 선언
    중첩클래스라고도 함
    종류
        인스턴스 내부 클래스
        static 정적 내부 클래스
        local 지역 내부 클래스
        anonymous 익명 내부 클래스 (제일 많이 쓰임)
        UI 이벤트처리, 스레드 객체 ..등 단발성 이벤트 처리에 자주 애용된다.
        익명 클래스는 전혀 새로운 클래스를 익명으로 사용하는 것이 아니라, 이미 정의되어 있는 클래스의 멤버들을 재정의 하여 사용할 필요가 있을때 그리고 그것이 일회성으로 이용될때 사용하는 기법이다.
        즉, 익명 클래스는 부모 클래스의 자원을 일회성으로 재정의하여 사용하기 위한 용도 인 것이다.



인스턴스 내부 클래스 instance inner class
    내부적으로 사용할 클래스를 선언(private 권장)
    외부 클래스가 생성된 후 생성됨
    OutClass outClass = new OutClass();
    OutClass.InClass inClass=OutClas.new InClass();

정적 내부 클래스 static inner class
    외부 클래스 생성과 무관하게 사용 가능
    정적변수 정적메서드 사용

지역 내부 클래스 local inner class
    지역 변수와 같이 메서드 내에서 정의하여 사용하는 클래스
    메서드 호출이 끝나면 메서드에 사용된 지역변수의 유효성은 사라짐
    메서드 호출 이후에도 사용해야 하는 경우가 있을 수 있으므로 지역 내부 클래스에서 사용하는 메서드의 지역변수나 매개변수는 컴파일러가 final로 선언.

익명 내부 클래스 Anonymouse Inner Class
    이름이 없는 클래스
        지역내부클래스 이름은 실제로 호출 되는 경우가 없어서 더 심플하게 하고자 만들어짐
    클래스의 이름이 생략
        하나의 인터페이스나 하나의 추상클래스를 구현하여 반환해야 할 때 쓰게 됨
    android widget의 이벤트 핸들러에 활용됨
