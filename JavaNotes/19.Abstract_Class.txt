1. 추상클래스 (Abstract Class)
    상위 클래스 정의 시 설계시 사용.
    메서드 선언만 해두고 구현부는 비워둠 ->메소드 오버라이딩을 강제함.
        -메서드 선언 declaration 
            반환타입, 메서드 이름, 매개변수로 구성
                abstract String getFood();
        -메서드 정의 definition
            메서드 구현과 같은 의미 implementation
            {}는 구현부
    
    추상클래스는 new 할 수 없음(객체화 할 수 없다)
      인터페이스의 default메서드와 유사함
        대신 추상메서드는 객체변수, 생성자, private 메서드 등을 가질 수 있다.
        static을 명시적으로 선언해야 함.
      
2. 추상 클래스 구현
    1. 클래스와 메서드에 abstract 
    2. 하위클래스가 상속하여 구현. 
    
3. 템플릿 메서드 패턴
    템플릿 메서드 정의
        추상메서드나 구현된 메서드를 활용하여 코드의 흐름(시나리오, 알고리즘)을 정의하는 메서드
        final로 선언하여 하위 클래스에서 재정의 할 수 없게 함
        프레임워크에서 많이 사용되는 설계패턴
        추상 클래스로 선언된 상위 클래스에서 템플릿 메서드를 활용하여 전체적인 흐름을 정의
        

  4. final 
    사용위치
        1)변수 : 상수
        2)메서드 : 오버라이딩 금지
        3)클래스 : 상속이 안됨
