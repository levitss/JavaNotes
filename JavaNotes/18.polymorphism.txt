다형성(polymorphism)
    하나의 객체가 여러 타입을 가질 수 있는 것
    같은 코드에서 다른 결과가 나옴
    정보은닉encapsule, 상속과 더불어 oop의 가장 큰 특징 중 하나
    유연

2. 다형성의 장점
    기존 요구사항을 추가하는 경우 유연한 대처 가능
    상속과 메서드 재정의를 활용하여 확장성 있는 프로그램을 만들 수 있음
    
3. 클래스간의 협력
    1)is a 관계 (inheritance)
        일반적인 개념->구체적인 개념
        상속은 클래스간의 결합도가 높다.->상위 클래스의 수정이 많으면 하위 클래스에 영향
        계층구조가 복잡하거나 hierarchy(계층구조)가 높으면 좋지 않다
    2)has a 관계(coposition)
        클래스가 다른 클래스를 포함하는 관계
        코드 재사용의 가장 일반적인 방법
            -Student 가 Subject를 포함하는 관계
            Library를 구현할 때 Arraylist 생성하여 사용
        상속 x
4. 다운캐스팅
    1)업캐스팅이 된 경우에만 가능하다.(축소되지 않음)
    2)하위 클래스로의 형변환은 명시적으로 해야 함
    Customer cl=new Vip(); 묵시적인 업캐스팅
    Vip vcl=(Vip) cl;  명시적인 다운캐스팅
    3)instanceof를 이용하여 인스턴스의 type 체크
